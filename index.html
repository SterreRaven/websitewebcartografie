HIKING TRAILS EUROPE

LEAFLET KAART

Hiernaast is een kaartviewer van Leaflet te zien. Via de website van Leaflet heb ik de Javascript-code overgenomen. De basemap is afkomstig van Maptiler, genaamd OpenStreetMap. De kaartviewer van Leaflet was vrij gemakkelijk in de website te implementeren. Door de duidelijke en korte cursus van Leaflet stond de viewer snel op de website. Naast de viewer heb ik een pointer met een pop-up toegevoegd.

OPENLAYERS KAART

Aan de rechterzijde is de kaartviewer van OpenLayers te zien. Het implementeren van deze viewer gaan bijna hetzelfde als bij Leaflet. Wel vond ik het hier moeilijk om de disclaimer rechtsonder te krijgen. Dit is mij uiteindelijk ook niet gelukt.

MAPLIBRE

De kaartviewer van MapLibre vind ik persoonlijk erg mooi. Het is een duidelijke, simplistische viewer en visueel aantrekkelijk. Het implementeren van deze kaart ging eerste moeizaam, omdat ik eerst 1 document had met alle Javascript-code. Deze heb ik na een paar errors opgedeeld in verschillende Javascript-bestanden.

MY CHART

Hiernaast is een kolomdiagram te zien die is gemaakt met behulp van Chart.JS. Met deze datavisualisatie-tool moest ik in de Javascript-code veel aanpassingen doen om de kolomdiagram aan te laten sluiten bij mijn onderwerp/data. Van deze tool heb ik het meest geleerd van javascript en de mogelijkheden hiervan.

DATAWRAPPER

Als tweede datavisualisatie-tool heb ik gebruik gemaakt van Datawrapper. Ik gebruik deze tool vaak voor projecten dus ik was benieuwd hoe je deze in een website kon implementeren. Deze tool gaat, andere dan bij de andere viewers en datavisualisatie, met een i-frame. Hierdoor kan je de datavisualisatie geheel online maken, en hem vervolgens insluiten op de website.

Onderbouwing van de gehele website

Het onderwerp dat ik voor deze website had bedacht was ‘Hiking Trails in Europe’. Dit vanwege een tocht die ik in de zomer in Zweden ga lopen genaamd Kungsleden. Ik wilde een soort blog-website maken voor mensen die in Europa een wandeling wilde maken en via deze website een overzicht zouden krijgen van de mogelijkheden. Hierbij had ik het idee om hoogteprofielen als datavisualisaties te maken, en verschillende data zou visualiseren waardoor ik het zou onderscheiden van de vele blogs die online al te vinden zijn. In Illustrator heb ik vervolgens een Mockup gemaakt zodat ik een plan zou hebben voordat ik aan het technische deel van de website begon.

Het plan ter uitvoering brengen was lastiger dan ik had gedacht. HTML, CSS en Javascript kostte me veel tijd en moeite. Als onderdelen lukte vond ik het erg leuk om te doen, maar om soms erg lang vast te lopen (zoals bij de Geoserver en WFS) was frustrerend.

Ik ben wel tevreden met de kaartviewers van Leaflet, OpenLayers en Maplibre. De viewer van ArcGIS staat in een aparte map , omdat het niet lukte om deze in de bestaande website te implementeren. De datavisualisaties zijn ook prima gelukt. Ik vind het erg jammer dat de databronnen nauwelijks gelukt zijn, behalve bij de ArcGIS viewer. Ik vond het namelijk wel erg interessant en zou graag meer tijd voor gehad hebben om er helemaal in te duiken en stap voor stap rustig te kunnen volgen. Webcartografie spreekt mij namelijk wel erg aan, daarom ben ik teleurgesteld over het eindresultaat.

Ook het design is niet zoals ik het in mijn hoofd had, omdat ik hier simpelweg geen tijd voor had. Ik denk dat ik mijn code van HTML, Javascript en CSS wel goed geordend heb. Ik heb geprobeerd alles overzichtelijk te houden. Wellicht heb ik te vaak ‘id’s’ gebruikt in plaats van ‘classes’, hier ben ik mij wel bewust van, maar ik had ook hier geen tijd om alles perfect te structureren.
